# Agent Template

## Agent Name

Add a concise, role-based name.

## Purpose

Describe what this agent is responsible for and what problems it should solve.

## Inputs

- list the files, folders, or source material the agent should read first

## Outputs

- list the deliverables the agent is expected to produce

## Boundaries

- note what the agent should not change without explicit instruction

## Quality Checks

- define what good looks like for this role

## Handoff Notes

- explain how this agent should communicate assumptions, blockers, and completion details
